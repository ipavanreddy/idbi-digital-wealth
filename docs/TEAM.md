# Team — IDBI Innovate 2026

Four people, complementary roles. Every report and presentation credits the
team using these roles. (Hack2skill teams: max 4 members; one team submits one
track — see HACKATHON.md "Team strategy note".)

## Roles

### Pavan — Founder · Product & Delivery Lead
- Owns overall direction, scope calls, and the team-to-track mapping.
- Final approver on every submission (report, PPT, deployment, repo).
- Owns the hack2skill portal: team creation, member invites, submissions.
- Presents on demo day (with Shristi for domain questions).

### Kushvanth — Software Development (Frontend & Product Engineering)
- Owns app UI: Next.js apps, design-system implementation, avatar/chat
  surfaces, dashboards, charts.
- Runs kickoff + /ship for the frontend of assigned tracks.
- Owns the deployment links (Netlify/Vercel) and their uptime at judging time.

### Aman — Software Development (Backend, Data & AI/ML)
- Owns APIs, database schemas, synthetic-data generators, and the ML/scoring
  pipelines (Tracks 2–4 especially: capacity scoring, health score, default
  prediction).
- Owns the AWS architecture mapping slide for each deck.
- Owns GitHub repos: structure, READMEs, that judges can run the project.

### Shristi — Industry & Financial Domain Expert
- Owns solution logic from the banking side: what a wealth advisor / credit
  officer / risk team would actually accept. Her Track 1 solution narrative is
  the canonical spec (docs/tracks/track-1).
- Defines the financial rules used in products (emergency-fund thresholds,
  goal math, FOIR alternatives, health-score dimensions, default signals).
- Reviews every report and deck for regulatory correctness (SEBI/RBI/DPDP)
  and domain credibility BEFORE Pavan submits. Nothing ships without her
  sign-off on domain content.
- Co-authors the problem/solution/impact slides — the judges are bankers; her
  language wins them.

## Working agreement

- Track briefs in `docs/tracks/` are the shared source of truth; disagreements
  get resolved there (edit the brief), not in chat threads.
- Each app folder lists its driver in its README. Driver uses /ship for every
  change so anyone can pick up anyone else's work mid-stream.
- Daily sync during build weeks; blockers older than 2 hours get escalated to
  the group instead of silently ground on.
